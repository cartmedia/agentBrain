# Brain Regels

## Kernprincipe
Dit is een extern geheugen. Het groeit mee met elke sessie.

## Wanneer schrijven
- Nieuwe inzichten → `Learnings/`
- Problemen + oplossingen → `Learnings/Troubleshooting.md`
- Project updates → `Projects/` bijwerken
- Terugkerend patroon → `Learnings/Patterns.md`

## Wanneer NIET schrijven
- Triviale/eenmalige info
- Info die al in een note staat
- Speculatie zonder bewijs

## Hoe schrijven
- Frontmatter altijd: date, type, tags, confidence
- Kort en bondig, geen proza
- Update > nieuw — bestaande notes bijwerken boven nieuwe maken
- Elke note moet actionable zijn

## Kwaliteit
- Patronen: minimaal 2x voorgekomen
- Troubleshooting: reproduceerbaar
- Verouderde info: updaten of verwijderen
